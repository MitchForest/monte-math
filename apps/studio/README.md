# @monte/studio – Knowledge Graph & Authoring Playground

The studio app is the internal tool for curriculum teams. It visualizes the math knowledge graph and previews interactive lesson scripts while sharing the same contracts as the player.

## Responsibilities

- Load skills/prerequisites from the API and render them via Pixi + Dagre.
- Allow curriculum authors to inspect relationships and navigate stages.
- Reuse the same UI/design system (Tailwind v4 + shadcn/ui) as the player for parity.
- Remain contract-bound: all reads/writes go through `@monte/api-client`.

## Stack Overview

| Concern      | Library                                        | Notes                                                |
|--------------|------------------------------------------------|------------------------------------------------------|
| Build        | Vite + TypeScript                               | Same toolchain as web                               |
| Data Fetch   | Zustand store + `apiClient`                     | Store wraps API list/get requests                    |
| Visualization| PixiJS + Dagre                                  | Layout + canvas graph rendering                      |
| UI           | Tailwind v4, shadcn/ui, lucide icons            | Components under `src/components/ui`                 |

## Integration Boundaries

- Consumes `@monte/api-client` for RPC calls; never touches the database or content files directly.
- Shares schemas from `@monte/shared` and relies on the contract definitions for types.
- Intended to add authoring mutations in future—those must be added to the contract before use.

## Running Locally

```bash
pnpm dev:studio       # http://localhost:3002
```

Ensure the API (`pnpm dev:api`) is running first so the store can load the graph.

## Folder Guide

- `src/components/knowledge-graph/` – Pixi canvas + detail panel.
- `src/components/lessons/` – Lightweight lesson script editor.
- `src/stores/skills-store.ts` – Zustand store fetching via API.
- `src/styles/globals.css` – Tailwind token layer generated by shadcn.
- `src/lib/orpc-client.ts` – API client setup (custom header namespace).

When building new authoring features, extend the shared contract (`@monte/shared`) and let the API implement the changes before wiring them into the store.
